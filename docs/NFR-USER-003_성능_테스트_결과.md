# NFR-USER-003 사용자 API 성능 테스트 결과

## 1. 테스트 목적

현재 구현된 사용자 및 인증 API가 정상 부하에서 다음 성능 기준을 충족하는지 확인한다.

- API별 p95 응답 시간 3,000ms 이하
- 요청 성공률 99% 이상
- HTTP 5xx 오류율 1% 미만
- 동시 사용자 100명

## 2. 테스트 환경

| 항목 | 내용 |
| --- | --- |
| 테스트 일자 | 2026-07-17 |
| API 서버 | FastAPI / Uvicorn |
| 서버 워커 | 1 |
| 데이터베이스 | MySQL |
| ORM | SQLAlchemy 2.0 AsyncSession |
| 비밀번호 해싱 | pwdlib Argon2 |
| 부하 테스트 | Locust 2.45.0 |
| 동시 사용자 | 100명 |
| 사용자 증가율 | 초당 10명 |
| 지속 테스트 | 주요 시나리오별 10분 |
| 테스트 환경 | 로컬 개발 환경 |

## 3. 성능 개선 내용

- FastAPI 요청 처리 시간 측정 미들웨어 추가
- 3초 초과 요청 warning 로그 적용
- SQLAlchemy `echo` 로그 비활성화
- 비동기 DB 커넥션 풀 설정
- Argon2 검증 및 해싱을 thread pool에서 실행
- 비밀번호 변경의 불필요한 Argon2 검증 제거
- 회원 정보 수정의 불필요한 `refresh()` 제거
- 사용자 목록 조회 컬럼 제한 적용
- 사용자 목록 페이지네이션 적용
- DB 저장 실패 시 rollback 처리

현재 구현된 모든 사용자·인증 API가 `NFR-USER-003`의 성능 목표를 충족했다.

가장 높은 p95 응답 시간은 회원가입 API의 420ms이며, 비밀번호 변경 API는 Argon2 검증 및 해싱을 포함한 상태에서 p95 280ms를 기록했다.

## 6. 테스트별 참고사항

- 로그인, 토큰 갱신, 사용자 목록, 마이페이지 조회는 혼합 시나리오로 10분간 측정했다.
- 회원 정보 수정은 사용자별 독립 계정으로 10분간 측정했다.
- 비밀번호 변경은 사용자별 독립 계정을 사용하고 요청 간격을 20~30초로 설정하여 10분간 측정했다.
- 권한 변경과 로그아웃은 혼합 시나리오로 10분간 측정했다.
- 회원가입은 테스트 계정 100개를 생성하는 100명 burst 방식으로 측정했다.
- FastAPI 서버와 Locust가 동일한 로컬 장비에서 실행되어 운영 환경 결과와 차이가 있을 수 있다.
- 회원 탈퇴 API는 현재 애플리케이션에 구현되어 있지 않아 테스트 대상에서 제외했다.
