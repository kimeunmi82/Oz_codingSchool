## 1일차 팀 작성 규칙
 
### 문서 목적

이 문서는 흉부 X-Ray 이미지 기반 폐렴 판독 모델을 활용한 폐렴 환자 관리 백오피스 프로젝트를 팀원들이 동일한 기준으로 개발하기 위한 협업 규칙을 정의한다.

팀원 간 코드 충돌, 중복 구현, 환경 차이, 모델 추론 결과 불일치를 최소화하고 누구나 같은 환경에서 프로젝트를 실행할 수 있도록 하는 것을 목표로 한다.

### 기본 협업 원칙
- 맡은 업무는 기간안에 개발한다.
- 팀원 간의 배려를 하고 눈치보지 않는다.
- 개발한 내용은 서로 공유하고 크로스 테스트 진행한다.
- 매일 15:00 이후 자습시간에 중요한 회의 및 개발 진행
- 특별한 일정이 있지 않은 경우 매일 20:00~22:00 모여서 개발 진행한다.
- 구현되지 않은 기능을 완료된 것처럼 문서에 작성하지 않는다.
- 확인되지 않은 모델 성능이나 기능은 추측해서 기록하지 않는다.
- 기능을 개발하기 전에 담당 파일과 인터페이스를 팀원에게 공유한다.
- 다른 팀원이 작성한 코드를 임의로 대규모 수정하지 않는다.
- 문제가 발생하면 오류 메시지, 실행 환경, 재현 방법을 함께 공유한다.
- 복잡한 구조보다 현재 프로젝트 기간 안에 재현 가능한 구조를 우선한다.
- 코드뿐 아니라 실행 방법, 환경 변수, 테스트 방법도 함께 관리한다.
