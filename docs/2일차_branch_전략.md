# Git 브랜치(Branch) 완벽 이해하기

## 1. 브랜치란 무엇인가요?
브랜치(Branch)는 직역하면 **'나뭇가지'**라는 뜻입니다.
소프트웨어를 개발할 때, 하나의 프로젝트 안에서 여러 사람이 각자의 기능을 따로 만들거나, 
완성된 버전을 유지하면서 동시에 새로운 기능을 시도해볼 수 있도록 하는 **'독립된 작업 공간'**입니다.

## 2. 왜 브랜치를 사용하나요?
*   **안전한 개발:** 메인 코드(`main`)를 건드리지 않고 새로운 기능을 만들 수 있습니다. 실수해도 메인 코드는 안전합니다!
*   **동시 작업:** 여러 명의 개발자가 동시에 서로 다른 기능을 개발할 수 있습니다.
*   **버전 관리:** 언제든 특정 브랜치로 돌아가서 과거의 상태를 확인할 수 있습니다.

## 3. 브랜치의 핵심 개념
*   **`main` (또는 master) 브랜치:** 프로젝트의 얼굴입니다. 항상 잘 돌아가는 최종 완성본이 저장되는 가장 중요한 길입니다.
*   **기능별 브랜치 (Feature Branch):** `login-page`, `add-user-api`처럼 내가 만들고 싶은 기능의 이름을 따서 만드는 보조 길입니다.
*   **Merge (병합):** 보조 길에서 만든 기능을 다시 메인 길(`main`)로 합치는 과정입니다.

## 4. 비유로 이해하기
> **"나무 한 그루(프로젝트)가 있는데, 메인 줄기는 그대로 두고 나뭇가지를 뻗어서 
> 그 끝에 새로운 열매(기능)를 맺는 것과 같습니다."**

1.  메인 줄기(`main`)에서 작업하다가...
2.  새로운 기능이 필요하면 가지(`feature-branch`)를 쳐냅니다.
3.  가지 위에서 열심히 기능을 구현합니다.
4.  기능이 완성되면 다시 메인 줄기(`main`)에 가지를 붙입니다(Merge).

## 5. 실무에서의 흐름
1.  `git checkout -b 새로운-기능-이름`: 새로운 가지를 만들고 그곳으로 이동합니다.
2.  열심히 코딩하고 저장(`git commit`)합니다.
3.  `git checkout main`: 다시 메인 줄기로 돌아옵니다.
4.  `git merge 새로운-기능-이름`: 만든 기능을 메인 줄기에 합칩니다.

---
*팁: 너무 많은 브랜치를 만들면 관리가 힘들어지니, 기능 하나가 끝날 때마다 메인에 합치고 브랜치를 삭제하는 습관을 들이는 게 좋습니다!*
